# Projects: Web Application Security

Hands-on projects demonstrating web vulnerabilities and secure coding practices.

---

## Projects Included

### 1. Input Sanitization Demo (`input_sanitization_demo.py`)
Demonstrates the importance of input validation and output encoding:
- SQL injection simulation
- XSS (Cross-Site Scripting) filtering
- Command injection prevention
- Before/after secure coding examples

**How to run:**
```bash
python input_sanitization_demo.py
```

### 2. Vulnerable Web App Simulator (`vulnerable_app_simulator.py`)
A command-line simulation of common web application vulnerabilities:
- Broken authentication
- Insecure direct object references
- Session management flaws
- With explanations of how to fix each issue

**How to run:**
```bash
python vulnerable_app_simulator.py
```

### 3. Session Security Demo (`session_security_demo.py`)
Demonstrates secure vs insecure session management:
- Predictable session tokens
- Session fixation
- Secure token generation
- Session timeout handling

**How to run:**
```bash
python session_security_demo.py
```

---

## Learning Objectives

- Understand how common web vulnerabilities work
- Learn to validate and sanitize user input
- Recognize insecure session management patterns

## Requirements

- Python 3.x (standard library only)
